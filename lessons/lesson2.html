Conditionals and Loops

Conditional Statements

do it... conditionally


var x = 3;
if (x === 3) {
  console.log("X is equal to three!");
} else {
  console.log("X is not equal to three!")
}

Conditionals may look like this:


if (expression) console.log("This is executed conditionally");

Or this:


if (expression) {
  // some conditionally executed code
}

Or this:


if (expression) {
  console.log("This is executed if the expression is true");
} else {
  console.log("This is executed if the expression is not true");
}

Let's talk about truth

Javascript has special types for dealing with tests. These types are called boolean types. They are represented in your code by the keywords true and false


if (true) {
  console.log("This will always execute");
}

if (false) {
  console.log("This will never execute");
}

Truthiness

Comparison operators like ===, !==, >=, <= evaluate to either true or false.

Switch Statements


switch (aBeatle) {
  case "John":
    console.log("He's dead");
    break;
  case "Paul":
    console.log("Not dead yet!");
    break;
  case "George":
    console.log("Dead. Dead. Dead.");
    break;
  case "Ringo":
    console.log("Not dead, I guess?");
    break;
  default:
    console.log("That's not a beatle!");
}

Loops! Loops! Loops!

Loops allow you to execute the same code over and over until some condition is met

For loops


        for (var i = 1; i !== 11; i = i + 1) {
          console.log("This is loop number " + i);
        }
  • i is initialized at 1;
  • The test is run. The loop will execute so long as it is true
  • i is incremented by 1
  • This will continue until the test returns false

While loops


          var i = 0;
          while (i < 10) {
            console.log("The current value of i is: " + i);
            i += 1;
          }

while loops are a little simpler than for loops. The statement only takes one paramenter: the conditional test. This means you have to handle any required initialization before the loops runs, and any incrementing that needs to happen has to happen within the body of the loop itself.

While loops


        var i = 1337;
        while (i !== 1337) {
          console.log("The code within this loop will never run");
        }

The while loop's test is run before the first time the loop code executes, thus it is possible that the loop may not run at all.

Do-While Loops


            var i = 1;
            do {
              console.log("Potato!");
              i += 1;
            } while (i < 10);
  • do-while loops are rather a lot like for loops
  • The main difference is that a do-while loop will always execute the loop code at least one time
  • This is useful when you have a block of code you know needs to run once, and maybe more times

Do-While Loops


var i = 1000000;
do {
  console.log("This code will run once no matter what
               condition follows the do block!"); 
} while (i < 10);
  • Notice that a semicolon follows the while conditional

Assignment

write some code that does the following:

  • declare an arbitrary string of arbitrary length
  • uses a loop and a condition (of some sort)
  • prints out the string, reversed and with its vowels deleted

A possible solution


    var myName = "It's a beautiful day. Don't let it get away!";
    var output = "";

    for (var i = myName.length - 1; i >= 0; i--) {
        switch (myName[i]) {
            case 'a': case 'e': case 'i': case 'o': case 'u':
            case 'A': case 'E': case 'I': case 'O': case 'U':
                break;
            default:
                output += myName[i];
        }
    }
    console.log(output);
